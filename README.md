## Кейс со смены Т-банка и Сириуса по направлению рекомендательные системы

Задача: релизовать логику cross-sell для построения рекомендаций по продуктовой корзине в онлайн-супермаркете, используя classic i2i подходы и LLM подход, для сравнения качества рекомендаций этих двух пайплайнов

### Используемые данные

Рекомендации дополняют имеющиеся в корзине товары до рецептов, поэтому для обучения и дообучения моделей использовался датасет [https://www.kaggle.com/datasets/rogozinushka/povarenok-recipes](Поваренок). На прототипе решения (веб-сайт) представлен каталог Вкусвилла, очищенный от готовой продукции и других элементов, не являющихся элементами рецептов. 

### Структура репозитория

В различных ветках представлена работа различных участников команды (например, в моей ветке polina_branch лежит офлайн валидация различных classic i2i подходов для выбора лучшей модели и валидация llm моделей, а также работа с данными и поиском в прототипе решения). В ветке master находится само веб-приложение
